# 🎨 moku / デザインガイド (v1.0)

## 1. アプリ概要まとめ

**moku (モク)** は、プログラミング学習を「日常に溶け込む心地よい体験」として提供するモバイルアプリ。

### コンセプト

- **「日常に溶け込む、わたしだけのコード・アトリエ」**
- 雲の上にある静かで光の入るアトリエのような世界観
- 「習う」ではなく「嗜む（たしなむ）」新しい学習スタイル

### ターゲット

- 感度の高い 20 代女性層（Z 世代〜ミレニアル世代）
- プログラミング初心者、英語が苦手な人
- 韓国雑貨やカフェ巡りが好きで、スマホの中身もおしゃれにしたい人

## 3. 韓国カワイイ Web デザイントレンド 2025-2026

### 3.1 キーコンセプト

韓国のかわいい(귀여운) Web/UI デザインは、**ミニマリズム**と**やさしさ**の融合が特徴。

#### ドーパミンカラー × ニュアンスカラー

- 鮮やかすぎない「くすみパステル」が主流
- ラベンダー、ミントグリーン、ダスティピンク、クリームイエロー
- 目に優しく、長時間見ても疲れない配色

#### クレイモーフィズム (Claymorphism)

- 3D 感のある柔らかいボタン・カード
- 丸みを帯びたフォルム
- ソフトシャドウとマットな質感
- 粘土やマシュマロのような触り心地の演出

#### マイクロアニメーション

- スクロールトリガーの繊細なアニメーション
- ホバー時のふわっとした動き
- ローディング時のかわいい待機アニメーション
- 完了時の「ポン」という心地よいフィードバック

#### ミニマル × 余白美

- 情報を詰め込みすぎない余白（ホワイトスペース）の活用
- 1 画面に 1 メッセージの原則
- 視線の流れを意識したレイアウト

### 3.2 タイポグラフィ

- **丸みのあるサンセリフ体**を基調
- 日本語: Noto Sans JP (Regular/Medium)
- 英語: Inter, Outfit, または Nunito
- 見出しは大きく、本文は読みやすい行間

### 3.3 アイコン・イラスト

- 線が細くシンプルなラインアイコン
- 塗りは控えめ、または淡いグラデーション
- キャラクターは顔がないか、ミニマルな点目（ドット顔）

## 4. moku デザインシステム

### 4.1 カラーパレット（Nuance Color）

```
┌─────────────────────────────────────────────────────────────┐
│  BASE                                                       │
│  ├── オフホワイト:     #FAFAFA  (メイン背景)                │
│  └── フォギーブルー:   #EEF2F5  (サブ背景・カード)          │
├─────────────────────────────────────────────────────────────┤
│  ACCENT                                                     │
│  ├── ラベンダーグレー: #E0DDE6  (メインアクセント)          │
│  ├── セージグリーン:   #D8E2DC  (成功・完了)                │
│  └── ダスティピンク:   #EBD4D4  (注意・エラー)              │
├─────────────────────────────────────────────────────────────┤
│  TEXT                                                       │
│  ├── チャコール:       #333333  (メインテキスト)            │
│  └── グレージュ:       #8F8C8A  (サブテキスト)              │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 コンポーネントスタイル

#### ボタン

- 角丸: 16px 以上の大きめ radius
- ソフトシャドウ: `0 4px 12px rgba(0, 0, 0, 0.06)`
- タップ時: 軽く沈む + 色が少し濃くなる

#### カード

- 角丸: 20px
- 背景: #EEF2F5 または 白
- シャドウ: 控えめなドロップシャドウ
- ボーダー: なし、または 1px の薄いグレー

#### 入力フィールド

- アウトライン: 1px グレー、フォーカス時ラベンダー
- プレースホルダー: グレージュ
- 角丸: 12px

### 4.3 アニメーションガイドライン

| 種類         | 継続時間    | イージング                   |
| :----------- | :---------- | :--------------------------- |
| マイクロ     | 150-200ms   | ease-out                     |
| 画面遷移     | 300-400ms   | cubic-bezier(0.4, 0, 0.2, 1) |
| ローディング | 1000-1500ms | linear (ループ)              |

## 5. キャラクター「Rumi (ルミ)」

### デザインコンセプト

- 雲 × マシュマロ × 抽象的なフォルム
- 顔は点だけのミニマル表現（または顔なし）
- 先生ではなく「ルームメイト」的存在
- 感情表現は控えめ（形や色の変化で表現）

### 表情パターン

| 状態   | 表現方法                       |
| :----- | :----------------------------- |
| 通常   | ふわっと浮いている             |
| 喜び   | 少し大きくなる、色が明るくなる |
| 励まし | 横にゆらゆら                   |
| 眠い   | 少し縮む、色がくすむ           |

---

## 6. 画面デザイン方針

### ホーム画面

- Rumi がくつろぐメインビジュアル
- 「今日のワーク」カードを 1 つ表示
- 余白を十分に確保したレイアウト

### レッスン画面

- 雑誌の目次のようなリストデザイン
- 進捗はドットやラインで控えめに表示
- すごろく風の派手なデザインは避ける

### アトリエ画面

- 集めた「タネ」で育てた植物を鑑賞
- プリセットの模様替えのみ（手間をかけさせない）
- 窓から見える空の色が学習継続で変化
