# ☁️ moku / 要件定義書 (v2.1 JP-Style)

## プログラミング学習アプリ 要件定義書

---

## 1. プロジェクト概要

### 1.1 アプリ名

**moku (モク)**

### 1.2 コンセプト

**「日常に溶け込む、わたしだけのコード・アトリエ」**

頑張って勉強する場所ではなく、好きな音楽を聴くように、リラックスしてコードに触れる場所。
「プログラミング＝英語ばかりで怖い」という壁を取り払い、
ミニマルで心地よい空間で、気づけば生活の一部になっているような体験を提供する。
「習う」のではなく、「嗜む（たしなむ）」ような新しい学習スタイル。

### 1.3 ターゲットユーザー

- 20 代を中心とした、感度の高い女性層（Z 世代〜ミレニアル世代）
- 「プログラミングは難しそう」「英語が苦手」と感じている初心者
- 韓国雑貨やカフェ巡りが好きで、スマホの中身もおしゃれに保ちたい人
- キャリアアップはしたいけど、ガツガツするのは苦手な人

### 1.4 対応プラットフォーム

- iOS / Android (Flutter 製クロスプラットフォーム)

## 2. デザインと世界観

### 2.1 ビジュアルコンセプト

**「Morning Haze (朝もや)」**

雲の上にある、静かで光の入るアトリエ。
パステルカラーよりも彩度を落とした「グレイッシュパステル」や「ニュアンスカラー」を使用。
線画は極限まで細く、余白（Whitespace）をデザインの主役に。

### 2.2 カラーパレット (Nuance Color)

```
Base:
- オフホワイト: #FAFAFA（メイン背景）
- フォギーブルー: #EEF2F5（サブ背景・カード）

Accent:
- ラベンダーグレー: #E0DDE6（メインアクセント）
- セージグリーン: #D8E2DC（成功・完了）
- ダスティピンク: #EBD4D4（注意・エラー）

Text:
- チャコール: #333333（メインテキスト）
- グレージュ: #8F8C8A（サブテキスト）
```

### 2.3 キャラクター

**「Rumi (ルミ)」** - 空のパートナー

- 雲のような、マシュマロのような、抽象的なフォルム（あえて目鼻をはっきり描かない、または点だけのミニマル顔）
- 先生ではなく「ルームメイト」。横にいて、ただ一緒にコーヒーを飲んでくれるような存在。
- 感情表現は控えめに。ふわっと形を変えたり、色が変わったりして反応する。

## 3. 機能要件と用語定義

おしゃれさを保ちつつ、直感的にわかりやすい「カタカナ」「やさしい日本語」を採用。

### 3.1 学習フロー用語

| 画面表示       | 旧名称   | 内容                                         |
| :------------- | :------- | :------------------------------------------- |
| **ストーリー** | おはなし | 導入。短いエッセイのようなテキスト           |
| **ガイド**     | 解説     | 概念のヴィジュアル解説                       |
| **クイズ**     | クイズ   | シンプルな選択問題                           |
| **ワーク**     | 演習     | 実際に手を動かすコーディング                 |
| **完了**       | できたね | 派手すぎず、ハンコをポンと押すような完了画面 |

### 3.2 ゲーミフィケーション（ライフログ風）

「ゲーム」というより「生活の記録」に近い感覚に設計。

- **ドロップ (Drop)**

  - 経験値。学習するとビーカーや瓶に「知識のしずく」が溜まっていく表現。
  - 数字で「+10XP」と出すより、視覚的に水が溜まる心地よさを重視。

- **タネ (Seeds)**

  - 通貨。学習継続で手に入る。
  - ショップで消費するのではなく、アトリエに「植える」と新しいインテリア（観葉植物など）が育つ。

- **デイズ (Days)**
  - 連続学習日数（ストリーク）。
  - カレンダーにシンプルな「・」がつくだけのミニマルな表示。
  - 7 日続くと、アトリエの窓から見える空の色が変わる（朝焼け → 青空 → 夕焼けなど）。

### 3.3 コードエディタ "Canvas"

- **シンタックスハイライト:**
  - 目に優しい低コントラストな配色テーマ。
- **エラー表示:**
  - 赤色は使わない。「Oops,」「おっと」といった軽いトーンで、ダスティピンク色の波線を表示。
  - 難しい専門用語は使わず、「ここがつながってないみたい」のように Rumi が教えてくれる。

### 3.4 AI パートナー (Claude / Gemini)

- **Tone & Voice (口調):**
  - 落ち着いたトーン。敬語すぎず、タメ口すぎない「丁寧なフラット語」。
  - NG: 「すごい！天才！」「がんばろうね！」（熱すぎる）
  - OK: 「いい感じ。」「その書き方、スマートだね。」「ここは、こうするともっと綺麗かも。」

## 4. アプリ構成と画面

### 4.1 主要画面

1.  **ホーム (Home)**

    - Rumi がくつろいでいるメイン画面。
    - 「今日のワーク」がシンプルにカード表示されている。
    - 英語表記にする場合も、必ず日本語の補助を入れる（例: `Today (今日のワーク)`）。

2.  **レッスン (Lesson)**

    - 縦スクロールの学習リスト。
    - すごろくのような道ではなく、おしゃれな雑誌の目次（Table of Contents）のようなリストデザイン。

3.  **アトリエ (Atelier)**

    - 集めた「タネ」で育てた植物や、アンロックしたインテリアが見られる場所。
    - ユーザーは配置を細かく悩まず、プリセットの「模様替え」を楽しむ（手間をかけさせない）。

4.  **マイページ (Profile)**
    - 学習記録の統計。
    - SNS シェア用のカードは、ポラロイド写真風のデザインで生成。

## 6. 開発フェーズ

### Phase 1: MVP "Quiet Start" (3 ヶ月)

- **目標:** 「1 言語(Python)で、心地よくコードが書ける」
- Rumi の基本実装と、エディタ(Canvas)の作り込み。
- BGM 機能の実装（Lo-Fi Hip Hop などの作業用 BGM）。

### Phase 2: Interior Update (2 ヶ月)

- **目標:** 「アトリエを育てたくなる」
- 「タネ」と「インテリア」要素の実装。
- 学習を続けるとアトリエが少しずつ充実していく環境変化の実装。

### Phase 3: AI & Monetization (2 ヶ月)

- **目標:** 「困ったときに頼れる」
- AI チャットの実装。
- プレミアムプラン（BGM 追加、AI 無制限など）の導入。
